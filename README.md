# StrmTool

Jellyfin 插件，用于从 strm 文件中提取媒体技术信息（codec、分辨率、字幕），加速 strm 媒体文件的起播速度。

核心功能

1. **媒体信息提前提取**：在 strm 文件入库后立即向远程服务器请求并获取媒体技术信息（音视频编码、分辨率、字幕等）
2. **计划任务支持**：提供 `Extract Strm Media Info` 计划任务，支持手动触发和定时执行
3. **配置界面**：提供插件设置页面，可调整刷新延迟时间避免对远程服务器造成压力
4. **媒体库完整性**：确保 strm 文件入库后媒体库页面显示完整的音视频信息
5. **独立媒体探测**：使用底层 FFprobe API 直接探测媒体流信息，完全避免调用元数据提供商（TMDB 等）和图像更新

## 版本

v1.1.0.0

## 环境要求

- .NET 9.0
- Jellyfin 10.11.0+（其他版本请自行测试）

## 安装方法

1. 在 Jellyfin 的 `plugin` 目录下新建文件夹 `StrmTool`
2. 将编译生成的 `StrmTool.dll` 放入该文件夹
3. 重启 Jellyfin 服务

## 使用方法

1. 进入 Jellyfin 后台 → 计划任务
2. 找到 `Strm Tool` 分类下的 `Extract Strm Media Info` 任务
3. 可手动运行或设置定时触发

## 配置说明

在插件详情页点击"设置"按钮，可以调整刷新延迟时间（毫秒），默认 1000ms。延迟用于避免对远程服务器造成压力。

## 更新说明

### v1.1.0.0 (2026-02-13)

**新功能**
- ✨ **自动提取媒体信息**：新入库 strm 文件自动在后台提取媒体信息，毫秒级响应
- 🔧 **自动提取开关**：在插件设置页面可启用/禁用自动提取功能
- 📊 **性能改进**：新文件无需手动运行计划任务，大大提升用户体验

**技术细节**
- 订阅 Jellyfin `ItemAdded` 事件自动检测新增 strm 文件
- 异步后台执行，不阻塞库扫描流程
- 延迟 1 秒后执行，确保 Jellyfin 完成初始化

### v1.0.0.3 (2026-02-13)

**改进内容**
- 🎯 **底层 API 优化**：改用 `IMediaEncoder.GetMediaInfo()` 直接调用 FFprobe 探测媒体流信息
- 🚀 **完全隔离**：完全避免调用 TMDB 等元数据提供商，不触发图像更新
- ✨ **性能提升**：简化处理流程，响应更快，更可靠
- 🔍 **独立操作**：媒体流探测不依赖元数据系统，独立工作

**技术细节**
- 使用底层 FFprobe API 而非高级元数据刷新接口
- 自动识别媒体源协议（HTTP、RTSP、FTP 等）
- 媒体流信息直接保存到数据库，无中间缓存依赖

## 注意事项

- 请根据使用的 Jellyfin 版本选择对应版本的插件（此版本仅支持 Jellyfin 10.11.0+）
- v1.0.0.3 相比之前版本不会调用任何第三方元数据服务，已有的元数据（标题、描述、海报等）不会被修改
